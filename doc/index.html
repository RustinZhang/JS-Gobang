h_heuristic.js

历史启发模块 简言之在评估节点时，如果该节点引发剪枝，说明先评估该节点可能增加搜索效率，故增加其权重。越靠近根节点引发剪枝的，权重越高。历史启发不像杀手启发那样需要依赖棋类知识。

t_table.js

置换表模块。采用深度搜索，会面临很多相同的局面，将评估过的局面存入置换表，下次遇到可以直接从表中调取得分。事实上抽象的数据结构是一个哈希表，这里采用 Object 实现，用 Map 会更直观一些，待有时间重构时可能会考虑更改。置换表采用的策略是用内存空间换取CPU计算时间。
